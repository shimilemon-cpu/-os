# 大喜利Pocket — アプリ構造・仕様書

## 概要
友達3〜5人で遊ぶAI搭載の大喜利アプリ。
AIが司会（お題出題）・審査員（3ペルソナで採点）・分析官（笑いの傾向を学習）の役割を担う。

---

## 技術スタック
- **フレームワーク**: Next.js 16 (App Router) / React 19
- **スタイル**: Tailwind CSS v4
- **DB / Auth**: Firebase (Firestore + Auth)
- **AI**: Anthropic Claude (Haiku)
- **デプロイ**: Vercel

---

## 画面一覧（9画面）

### 1. ログイン画面 `/auth/login`
- Googleでログインボタン（1つだけ）
- アプリ名・キャッチコピー表示

### 2. ルーム一覧 `/rooms`
- 参加中のルームカード一覧（ルーム名・人数・ステータス）
- 招待コード入力フォーム（6桁英数字）
- 「作成」ボタン

### 3. ルーム作成 `/rooms/new`
- ルーム名入力
- モード選択（リアルタイム / 非同期）の2択カード

### 4. 待合室 `/rooms/[id]`
- 招待コード大きく表示 + コピーボタン
- メンバー一覧（名前・準備OK/待機中）
- 「準備完了！」ボタン（一般メンバー）
- 「ゲームを開始！」ボタン（ホストのみ・全員準備OK後に有効）

### 5. 回答フェーズ `/rooms/[id]/game`
- カウントダウンタイマー（数字 + バー）
- お題テキスト（大きく中央表示）
- 回答入力テキストエリア
- 「回答する」ボタン
- 送信後：「✓ 回答を送信しました」確認表示

### 6. 投票フェーズ `/rooms/[id]/game/vote`
- カウントダウンタイマー
- お題テキスト（小さく）
- 回答カード一覧（A・B・C・D…のラベルで匿名表示）
  - 各カードに😂/🧠/🤯のリアクションボタン
  - 自分の回答はハイライト

### 7. ラウンド結果 `/rooms/[id]/game/result`
- 👑 MVP回答（最多票）ハイライト表示
- 全回答カード（票数付き）
- AI審査員3人の講評（王道👑・辛口🔪・カオス🌀）
  - 各審査員のスコア（0〜100点）とコメント
- 「次のラウンドへ」ボタン（ホストのみ）

### 8. 最終集計 `/rooms/[id]/summary`
- 🏆 優勝者（総得票1位）
- スコアランキング
- 各ラウンドのMVP一覧
- 「笑い分析を見る」ボタン
- 「結果をシェア」ボタン

### 9. 笑い分析 `/rooms/[id]/analysis`
- 統計数値（ラウンド数・総投票数）
- 反応傾向バーグラフ（😂面白い / 🧠うまい / 🤯狂ってる）
- AI審査員平均点バーグラフ
- 殿堂入り回答 TOP5

### 10. マイページ `/mypage`
- アバター・ニックネーム・メールアドレス
- 参加ルーム数
- テーマ切替（sepia / wine）
- アプリ説明

## 共通UI要素

### ボトムナビゲーション
- ルーム（ホームアイコン）
- 作成（＋アイコン）
- マイページ（人物アイコン）
- ※ゲーム中・集計画面では非表示

### カードコンポーネント（回答カード）
- 回答テキスト
- 匿名ラベル（A〜D）または作者名（結果公開後）
- 😂 / 🧠 / 🤯 リアクションボタン（票数付き）
- 自分の回答はアクセントカラーで縁取り

### タイマーコンポーネント
- 大きな数字
- 下部にプログレスバー
- 残り30秒で黄色、10秒で赤に変化

## ゲームフロー（リアルタイムモード）
```
ルーム作成 → 待合室（全員準備OK）
→ [ラウンド繰り返し（デフォルト5回）]
   AIがお題生成（90秒タイマー）
   → 回答入力 → 匿名一斉公開
   → 投票（45秒タイマー）
   → AI講評（3ペルソナ×全回答）
   → ラウンド結果
→ 最終集計 → 笑い分析
```
